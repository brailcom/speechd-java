Package {@link speechd.ssip} contains classes and interfaces implementing the client part of the SSIP protocol. Package {@code speechd.tests} contains unit tests for these classes.

SSIP Client

The {@link speechd.ssip.SSIPClient} class is the main entry point for communicating with SSIP servers. You create an instance of {@link speechd.ssip.SSIPClient }, it connects to server and you can use the various provided methods to send speaking commands, set synth parameters, etc.. We recommend taking a look on {@link speechd.ssip.SSIPClient} documentation.

Are also provided auxiliary classes and enumeration to easy {@link speechd.ssip.SSIPClient} usage and client interface.

Error handling

Error conditions are handled using java's exception handling mechanism. All thrown exceptions are subclasses of {@link speechd.ssip.SSIPException}, see {@link speechd.ssip.SSIPCommandException}, {@link speechd.ssip.SSIPDataException} and {@link speechd.ssip.SSIPCommunicationException}.

Low level SSIP communication

The {@link speechd.ssip.SSIPConnection} class is responsible for low level SSIP communication handling: sending commands, arcing responses, and all network negotiation. {@link speechd.ssip.SSIPClient} is implemented over an instance of {@code SSIPConnection}. You can use {@link speechd.ssip.SSIPConnection} directly if you want to use different commands from those {@link speechd.ssip.SSIPClient} provides, or implement those commands a different way. See also the {@link speechd.ssip.SSIPCommand} and {@link speechd.ssip.SSIPResponse} classes.

Event Handling

SSIP event handling is also supported, see the {@link speechd.ssip.SSIPEvent} class and {@link speechd.ssip.SSIPEventHandler} interface for documentation.

Logging activities

This code uses java standard logging api to log activities for debugging porposes, all classes log to descendents of the {@code speechd} logger. See the documentation for specific classes to know what loggers are used and the Java logging overview
